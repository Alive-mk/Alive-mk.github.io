GRPO (Group Relative Policy Optimization)

2026-02-02 23 min read # 实习

1. 核心概念与论文溯源 (Concept & Paper Roots)

  • 论文引用：

    • 起源论文：DeepSeekMath: Pushing the Limits of Mathematical Reasoning in Open Language Models (arXiv:2402.03300)
    • 爆发应用：DeepSeek-R1: Incentivizing Reasoning Capability in LLMs via Reinforcement Learning (arXiv:2501.12948)

  • 本质概括：

    GRPO 是一种去除了 Critic 网络（Value Function）的 PPO 变体，它通过对同一个问题采样一组（Group）回答，利用组内相对优势来替代传统的值函数基线（Baseline）。

  • 痛点解决：

    • 显存/计算开销大：传统的 PPO 需要维护一个与 Policy 模型等大的 Critic 模型（以及其优化器状态），显存占用几乎翻倍。GRPO 彻底抛弃了 Critic，大幅降低了 RL 训练的硬件门槛。
    • 基线估计方差：在数学推理等任务中，答案的对错通常是确定的，但生成路径多样。GRPO 利用“同一问题的多条采样路径”作为天然的基线（Baseline），比训练一个并不完美的 Critic 网络更能准确反映“在这个 Prompt 下，当前回答好不好”。

第一部分：深度数学解析 (The Mechanics)

GRPO 的数学核心在于它如何构建“基线”（Baseline）。在强化学习中，仅仅知道一个动作得到了多少分（Reward rrr）是不够的，我们需要知道这个分数是**“超常发挥”还是“失常发挥”**。

1. 优势函数 AiA_iAi​：去除了 Critic 的“相对论”

Ai=ri−mean({r1,...,rG})std({r1,...,rG})+ϵA_i = \frac{r_i - \text{mean}(\{r_1, ..., r_G\})}{\text{std}(\{r_1, ..., r_G\}) + \epsilon} Ai​=std({r1​,...,rG​})+ϵri​−mean({r1​,...,rG​})​

  • 传统 PPO 的做法：训练一个巨大的 Critic 网络 V(s)V(s)V(s) 来预测“在这个状态下，一般能得多少分”。优势 A=r−V(s)A = r - V(s)A=r−V(s)。这非常昂贵，因为 Critic 网络通常和 Policy 网络一样大。
  • GRPO 的做法：
    • Group Sampling：对于同一个问题 qqq，让模型生成 GGG 个不同的回答 {o1,…,oG}\{o_1, \dots, o_G\}{o1​,…,oG​}。
    • Baseline = Group Mean：我们将这 GGG 个回答的平均分作为基线。
    • 物理含义：如果 Ai>0A_i > 0Ai​>0，说明第 iii 个回答的表现优于这组回答的平均水平；反之则差于平均。
    • 方差控制：除以标准差 std\text{std}std 是为了归一化（Z-Score Normalization）。这消除了“题目难度”的影响——无论题目多难（导致全员低分）或多简单（导致全员高分），我们只关心哪个回答相对更好。

    • 深层原因：在强化学习中，模型很容易学会“作弊”（Reward Hacking）。例如，如果 Reward 是基于答案长度给分的，模型可能会无限重复废话。KL 惩罚迫使模型在探索高分的同时，保持语言的流畅性和逻辑性（即保持 SFT 学到的人类语言分布）。

第二部分：通俗易懂的解释 (The Intuition)

如果你在面试中需要快速向别人解释 GRPO，可以用下面这个**“考试与补习班”**的比喻：

场景设定

你是老师（优化算法），DeepSeek 模型是一个学生。你想提高他的解题能力。

1. 传统 PPO 方法（也就是带 Critic 的方法）

  • 学生做一道题，得 80 分。
  • 为了知道这 80 分算好还是坏，你必须聘请一个资深辅导员 (Critic Model)。
  • 辅导员看了一眼题目，说：“这题很难，一般人只能拿 60 分。”
  • 于是你知道：80 分属于超常发挥，优势是 +20。你会重点表扬学生。
  • 缺点：你需要花大价钱（显存、算力）养这个辅导员。

2. GRPO 方法（DeepSeek-R1 的方法）

  • 你辞退了辅导员（省了一半的钱）。
  • 你给学生一道题，让他连续做 6 遍 (Group Sampling)，生成 6 个不同的解题过程。
  • 结果出来了，分数分别是：[50, 50, 50, 80, 50, 50]。
  • 算出平均分是 55 分。
  • 逻辑：
    • 那个得了 80 分的解法，比平均分高，它是好的（Advantage > 0）。学它！
    • 那些得 50 分的解法，比平均分低，它们是相对差的（Advantage < 0）。少用这种解法！

总结

GRPO 的核心哲学就是：不依赖外部权威（Critic）的绝对打分，而是通过“平行宇宙”中的多次尝试，让模型在自己的尝试中“择优而从之”。 只要这一组尝试里有一个比较好，模型就能捕捉到这个信号并进化。

3. 手撕代码重点与实现 (Whiteboard Coding Focus)

在白板编程中，面试官不要求你写完整的 PPO Loop，但必须能够默写出 GRPO 的 Loss 计算逻辑。

核心逻辑（必须死记硬背）：

  1. 输入是 old_log_probs, new_log_probs, rewards。
  2. 维度操作：数据通常是 (Batch, Group_Size, Sequence_Length)。
  3. Advantage 计算：必须在 Group_Size 维度上做 Normalize。

Python

import torch
import torch.nn.functional as F

def grpo_loss(
    old_log_probs: torch.Tensor,  # Shape: (B, G, L) - 采样的旧策略概率
    new_log_probs: torch.Tensor,  # Shape: (B, G, L) - 当前策略概率
    rewards: torch.Tensor,        # Shape: (B, G)    - 每个采样的最终奖励
    epsilon: float = 0.2,         # PPO clip range
    beta: float = 0.01            # KL penalty coefficient
):
    """
    计算 GRPO Loss。
    B: Batch Size (number of queries)
    G: Group Size (samples per query)
    L: Sequence Length
    """
    
    # 1. 计算 Advantage (核心：组内归一化)
    # rewards 维度是 (B, G)，我们需要在 G 维度求均值和方差
    mean_rewards = rewards.mean(dim=1, keepdim=True)
    std_rewards = rewards.std(dim=1, keepdim=True)
    
    # 加上 1e-8 防止除零
    advantages = (rewards - mean_rewards) / (std_rewards + 1e-8)
    
    # 将 advantages 扩展到序列长度维度 (B, G, 1) -> (B, G, L)
    # 注意：通常 RL 训练中，Token 级别的 Advantage 可能需要 Mask 或衰减，
    # 但在简化的 GRPO 实现中，常将整条序列的 Advantage 视为相同。
    advantages = advantages.unsqueeze(-1).expand_as(old_log_probs)

    # 2. 计算重要性采样比率 (Ratio)
    # Log 空间相减等于概率相除: exp(log_new - log_old)
    ratio = torch.exp(new_log_probs - old_log_probs)

    # 3. PPO Clipped Objective
    surr1 = ratio * advantages
    surr2 = torch.clamp(ratio, 1 - epsilon, 1 + epsilon) * advantages
    ppo_loss = -torch.min(surr1, surr2).mean() # 负号因为我们要最大化目标

    # 4. KL Divergence Penalty (近似计算)
    # approx_kl = (old_log_probs - new_log_probs) # 简化写法，实际可能用 ref model
    # 这里假设 new_log_probs 包含 ref 约束，或者作为单独项处理。
    # 简单实现中，通常 explicitly 计算 ref_model 的 log_probs
    # kl_loss = beta * ((torch.exp(ref_log_probs) / torch.exp(new_log_probs))... )
    # 为保持代码简洁，这里略去 ref_model 的前向过程，仅展示 PPO 部分。
    
    return ppo_loss

# 工程细节 Note:
# 1. Masking: 真实代码必须应用 Attention Mask，只计算 Answer 部分的 LogProbs，忽略 Padding 和 Question。
# 2. Token-level Reward: 如果是 Process Reward Model (PRM)，advantages 会更复杂；
#    如果是 Outcome Reward，全序列共享一个 Advantage。

4. 面试高频考察点 (Interview Checkpoints)

Q1: GRPO 相比 PPO 最大的优势是什么？为什么它能在大模型推理中成功？

  • 满分回答：
    1. 资源效率：移除了 Critic 模型，节省了约 50% 的显存和参数更新开销，使得在有限资源下训练超大模型（如 DeepSeek-R1）成为可能。
    2. 基线稳定性：在推理（Reasoning）任务中，使用 Group Mean 作为 Baseline 比训练一个 Value Model 更稳健。因为推理题通常有明确的正确标准，组内比较（Group Relative）能自动适应不同难度的 Prompt（难的题大家都分低，相对分仍然能拉开差距）。

这是一个非常关键的面试题，它触及了 GRPO 能够取代 PPO 成为 DeepSeek-R1 核心算法的底层工程逻辑与数学直觉。

以下是针对这两个维度的深度解析与通俗解释。

第一部分：深度技术解析 (Deep Technical Analysis)

1. 资源效率：打破“显存墙” (Breaking the Memory Wall)

在 SFT（监督微调）之后的 RL 阶段，训练超大模型（如 70B 或 671B 参数）面临的最大瓶颈不是时间，而是显存（VRAM）。

  • PPO 的痛点（Critic 负担）：

    PPO 是一种 Actor-Critic 架构。为了计算优势函数 A(s,a)=r−V(s)A(s,a) = r - V(s)A(s,a)=r−V(s)，你不仅需要加载策略模型（Actor/Policy），还需要训练一个等大的价值模型（Critic/Value Function）。

    • 显存翻倍：假设 Policy 是 70B 参数，Critic 通常也是 70B。这就相当于显存占用直接 ×2\times 2×2。
    • 优化器状态爆炸：在使用 Adam 优化器时，每个参数需要维护 2 个状态（Momentum 和 Variance）。增加一个 70B 的 Critic，意味着显存中要多存 70B×2×4bytes70B \times 2 \times 4\text{bytes}70B×2×4bytes 的优化器状态。
    • 通信瓶颈：在分布式训练（如 ZeRO Stage 3）中，梯度的同步开销也随参数量翻倍。

  • GRPO 的突破：

    GRPO 彻底移除了 Critic 网络。

    • 它不需要神经网络来预测 V(s)V(s)V(s)，而是通过当前 Batch 内的统计数据（Mean/Std）直接计算优势。
    • 结果：显存占用几乎减半，通信开销大幅降低。这使得 DeepSeek 能够在同等硬件资源下，训练参数量更大的模型，或者显著增加 Context Length（推理长思维链需要极长的 Context）。

2. 基线稳定性：规避“价值评估难”的问题 (Robust Baseline)

这解释了为什么 GRPO 特别适合**推理（Reasoning）**任务，而不是普通的聊天任务。

  • PPO 的痛点（Value Estimation 极难）：

    在数学或代码推理中，Reward 通常非常稀疏（只有最后做对了才有分，中间步骤没分）。

    • 让 Critic 网络去预测“写到第 5 行代码时，最后能不能运行成功”，这是一个极其困难的回归问题。
    • 如果 Critic 预测不准（Underfitting 或 Overfitting），计算出的优势 AtA_tAt​ 就是充满噪声的垃圾信号，会导致 Policy 更新方向错误，甚至导致模型训练崩塌（Collapse）。

  • GRPO 的突破（Group Relative）：

    GRPO 使用 Ai=ri−mean(R)std(R)A_i = \frac{r_i - \text{mean}(R)}{\text{std}(R)}Ai​=std(R)ri​−mean(R)​。

    • 自适应难度（Adaptive Baseline）：
      • 遇到难题：Prompt 很难，模型采样 64 次全做错了，分数全是 0 或很低。此时 Mean 很低。如果有一次偶然做对了一点点，它的 Advantage 就会非常高（脱颖而出）。
      • 遇到简单题：Prompt 很简单，64 次全满分。此时 Mean 很高，std 很小。所有回答的 Advantage 趋近于 0。这相当于自动告诉模型：“这题太简单了，大家都会，不用刻意更新参数。”
    • 减少方差：这种基于 Monte-Carlo 的基线估计，虽然依赖采样，但在 Reward 确定的推理任务中，比训练一个半吊子的 Critic 网络要稳定得多。

第二部分：通俗易懂的解释 (The Plain Language Explanation)

你可以用**“体育课跳高测试”**来比喻这两种模式的区别：

场景

我们要训练一个运动员（模型）跳得更高。

1. PPO 模式：依赖“专业教练”

  • 机制：你需要花重金聘请一个专业教练 (Critic)。
  • 过程：运动员跳了一次。教练根据经验打分：“虽然你跳过去了，但我预测你这个姿势下次会失败，所以给你打低分。”
  • 问题 (资源)：教练工资很高（显存占用大）。
  • 问题 (稳定性)：如果教练水平很菜（Critic 没训练好），瞎指挥，运动员就会被带偏，越练越废。在复杂的数学推理中，由于步骤极其复杂，很难找到这种“全知全能”的教练。

2. GRPO 模式：依赖“小组内卷”

  • 机制：不请教练了。让运动员自己分身成 64 个人 (Group Sampling)，一起跳。
  • 过程：
    • 大家一起跳。有的跳了 1 米，有的跳了 1.5 米，有的跳了 2 米。
    • 算出平均成绩是 1.5 米。
    • 逻辑：那个跳 2 米的分身说：“我比平均水平高 0.5 米，我是对的，大家学我！”；那个跳 1 米的分身说：“我比平均水平低，我是错的，大家别学我。”
  • 优势 (资源)：省下了请教练的钱（省显存）。
  • 优势 (稳定性)：
    • 不管杆子有多高（题目多难），只要这 64 个分身里有一个相对较好的，大家就有学习的目标。
    • 不管杆子有多低（题目多简单），如果大家都跳过去了，就没有“相对优势”，大家就保持现状，不会瞎改动作。

一句话总结：

GRPO 之所以成功，是因为它承认在复杂的推理任务中，与其训练一个甚至可能不如模型本身的“裁判”（Critic），不如让模型通过“大规模并行尝试”来自己跟自己比，优胜劣汰。

Q2: GRPO 中的 Advantage 计算公式是什么？如果 Group Size = 1 会发生什么？

  • 满分回答：

    公式是 Ai=(ri−μG)/σGA_i = (r_i - \mu_G) / \sigma_GAi​=(ri​−μG​)/σG​。

    如果 Group Size = 1，均值 μG\mu_GμG​ 就是 r1r_1r1​ 本身，分子变为 0，标准差未定义（或为 0）。这意味着 AiA_iAi​ 无法计算或恒为 0，梯度消失，无法进行策略更新。因此 GRPO 强制要求 G>1G > 1G>1（通常 G=64G=64G=64 或更多）。

这是一个在面试中非常容易被忽略，但一旦问出就能瞬间区分“调包侠”和“原理懂行者”的陷阱题。

以下是针对该问题的深度数学推导与通俗解释。

第一部分：深度技术解析 (Deep Technical Analysis)

这个问题触及了 GRPO “相对优势” (Relative Advantage) 的数学定义本质。GRPO 是一种基于 Z-Score 标准化 的策略优化方法。

1. 数学推导：为什么 G=1 会导致训练瘫痪？

回顾 Advantage 的计算公式：

Ai=ri−μGσG+ϵA_i = \frac{r_i - \mu_G}{\sigma_G + \epsilon} Ai​=σG​+ϵri​−μG​​

其中：

  • μG=1G∑j=1Grj\mu_G = \frac{1}{G} \sum_{j=1}^G r_jμG​=G1​∑j=1G​rj​ （组内均值）
  • σG=1G∑j=1G(rj−μG)2\sigma_G = \sqrt{\frac{1}{G} \sum_{j=1}^G (r_j - \mu_G)^2}σG​=G1​∑j=1G​(rj​−μG​)2​ （组内标准差）

当 Group Size G=1G = 1G=1 时：

  1. 均值塌缩：

    样本集只有 {r1}\{r_1\}{r1​}。

    μG=11∑r1=r1\mu_G = \frac{1}{1} \sum r_1 = r_1 μG​=11​∑r1​=r1​

    均值直接等于该样本的奖励值。

  2. 分子归零 (The Null Numerator)：

    分子=r1−μG=r1−r1=0\text{分子} = r_1 - \mu_G = r_1 - r_1 = 0 分子=r1​−μG​=r1​−r1​=0

    这是最致命的一步。任何样本与自身的均值做差，结果必然为 0。

  3. 分母失效 (The Vanishing/Undefined Denominator)：

    σG=(r1−r1)2=0\sigma_G = \sqrt{(r_1 - r_1)^2} = 0 σG​=(r1​−r1​)2​=0

    虽然我们在代码中通常会加一个极小值 ϵ\epsilonϵ (epsilon) 防止除以零报错，但分母在数学意义上已经失去了衡量“离散程度”的能力。

  4. 最终结果：梯度消失 (Gradient Vanishing)：

    计算出的优势 A1=00+ϵ=0A_1 = \frac{0}{0 + \epsilon} = 0A1​=0+ϵ0​=0。

    在策略梯度（Policy Gradient）更新公式中：

    ∇θJ≈E[∑Ai∇θlog⁡πθ(o∣q)]\nabla_\theta J \approx \mathbb{E} \left[ \sum A_i \nabla_\theta \log \pi_\theta(o|q) \right] ∇θ​J≈E[∑Ai​∇θ​logπθ​(o∣q)]

    由于 AiA_iAi​ 恒为 0，∇θJ\nabla_\theta J∇θ​J 恒为 0。

    结论：无论模型生成了什么（哪怕是完美的答案），参数更新量永远为 0，模型权重纹丝不动，训练完全停止。

2. 统计学视角：为什么 G 需要尽可能大（如 64）？

GRPO 的本质是利用蒙特卡洛采样 (Monte Carlo Sampling) 来近似真实的奖励分布。

  • 如果 GGG 太小（例如 G=2G=2G=2 或 444）：采样的方差极大。由于偶然性，可能好答案被当作坏答案（如果另一个答案碰巧更好），或者基线波动剧烈，导致训练不稳定。
  • 如果 GGG 足够大（例如 G=64G=64G=64）：根据大数定律，组内的均值 μG\mu_GμG​ 能较好地代表“当前策略在该问题下的平均水平”。此时，AiA_iAi​ 才能准确反映出样本 oio_ioi​ 的真实质量。

第二部分：通俗易懂的解释 (The Plain Language Explanation)

你可以用**“没有参照物的比赛”**来比喻。

场景：跑步比赛

  • GRPO 的逻辑是：不看绝对时间（不关心你是跑了 10 秒还是 20 秒），只看排名。跑得比平均快就是好，跑得比平均慢就是差。

1. 当 Group Size = 1 时（一个人跑）

  • 你一个人在操场上跑了一圈。
  • 算法问：“你跑得算快还是慢？”
  • 你回答：“我跑出了我的平均速度。”（因为只有你一个人，平均值就是你现在的成绩）。
  • 比较结果：你比平均值快 0 秒，也慢 0 秒。
  • 算法的反应：既然你表现得“平平无奇”（Advantage = 0），既不值得表扬，也不值得批评。
  • 结果：算法什么都不做，你的跑步姿势一点都不会改变。你白跑了。

2. 当 Group Size > 1 时（一群人跑）

  • 你和你的 63 个分身一起跑。
  • 如果平均成绩是 15 秒，而你跑了 12 秒。
  • 比较结果：你比平均快 3 秒！
  • 算法的反应：“哇，这个姿势能跑得比平均快！加分！记住这个姿势！”
  • 结果：模型学到了东西，参数更新了。

总结：

GRPO 是一种**“比较学习法”**。如果没有同伴（Group Size = 1），就没有比较；没有比较，就没有优劣；没有优劣，模型就不知道该往哪个方向进化。

Q3: 为什么 GRPO 需要 KL 散度惩罚？

  • 满分回答：

    为了防止 Reward Hacking（模型通过生成乱码或走捷径来骗取高分）以及防止模型遗忘（Catastrophic Forgetting）原有的语言能力。KL 项约束策略 πθ\pi_\thetaπθ​ 不会偏离经过 SFT 的参考模型 πref\pi_{ref}πref​ 太远，保证输出的流畅性和可读性。

5. 场景与演进 (SOTA & Evolution)

  • 业务场景：
    • 强逻辑推理：数学解题（Math）、代码生成（Coding）。
    • 有明确 Ground Truth 的任务：可以通过编译器通过率、数学答案匹配来自动计算 Reward 的场景。
    • Self-Play / Self-Reasoning：DeepSeek-R1 的“Aha Moment”即产生于这种通过大规模采样和相对优化的过程中。
  • SOTA 演进：
    • 当前地位：GRPO 是目前 Open-Weight LLM（开源权重模型） 领域训练推理模型的最优解（SOTA），特别是 DeepSeek-R1 的成功证明了其在极低成本下能达到类似 OpenAI o1 的推理能力。
    • 变体与改进：
      • Process Reward (PRM) + GRPO：除了最终答案的 Reward，还在推理步骤中加入 Step-level 的 Reward 信号，进一步细化 Advantage 的计算。
      • Iterative GRPO：类似于 DeepSeek-R1 的流程，先做一轮 Cold Start SFT，再做 GRPO，再生成数据做 SFT，多轮迭代以提升数据质量。
